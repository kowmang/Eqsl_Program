1. Introduction

Database and Administration Program for EQSL Cards

The program is designed to import QSL cards downloaded from the eqsl.cc website using the "AC9HP Downloader program",
based on the fixed folder structure and file name structure of the image files,
and write them to an SQLite3 database, which holds your log in .adif format to crosscheck the incoming cards.
Individual cards you may receive per mail, can also be entered and added per hand.
A search function makes it easy to find individual or multiple cards (for awards, diplomas, etc.),
which can then be viewed or converted back to .png format via a export function in a specific folder.

The program is written in Python, and the GUI is created with QT Designer.

3. Overview of the windows

Here we will start with what all good manual readers want to know, which section or button does what.
After starting the program, you will see the main window with several buttons and options.
Only click "OK" to close the message and proceed. 3.1 Main Window

In the green section, on the top left you see the name of the database file currently opened.
Below in the red one, you have the main action buttons:
- File Menu: - Settings - Exit - Import Menu: - Single Card Import (maunell) - Bulk Card Import (from folder automatic) - Help Menu: - User Manual - About Eqsl Program On the left side in the blue section, you have the Image Preview.
On the right side in the magenta section, you have the Search and Export functions.
And at the bottom in the yellow section, you have the Database viewed.
For closing the program, just click on the Exit button in the File menu or on the red X, in the uppest right corner like in every windows system.
3.2 Settings Window

Here in the yellow section you can set the path to an existing SQLite3 database file,
via the "Select Button" or create a new one by clicking on "New DB Button". Then a window will open to browse to the desired folder and enter a name for the new database file.
In the middle at the blue section, you can set the path to your ADIF log file,
which is used to crosscheck incoming QSL cards with the "Select Button" and then import it into the Database with "Import Button". Attention: Program does not work without this file!!
At the bottom in the orange section, you can set the path to the folder,
where the exported QSL card images will be saved to with the "Select Button".
After setting all paths, click on "Cancel Button" to leave the window, changes will be automaticly saved. With the "Restore Button" you can clear the paths to programs defaults.
3.3 Single Card Import Window

Here you can import individual QSL cards, which you may have received per mail. You have to fill in the asked fields for the crosscheck with the database in the orange field.
Click on the "Select Button" in the magenta field to browse to the image file of the card. In the green field you have the "Import Button" to import the image into the database.
Click on "Cancel Button" to leave the window.
3.4 Bulk Card Import Window

Here you can import multiple QSL cards at once, which you have downloaded from eqsl.cc with the AC9HP Downloader program. In the red field are important infos for the file name syntax.
Click on the "Select Button" in the magenta field to browse to the folder, where the card images are stored.
In the green field you have the "Import Button" to import all images from the folder into the database.
Click on "Cancel Button" to leave the window.
3.5 Help Menu
In the Help Menu you can open this User Manual or view info about the Eqsl Program.
4. Start with the work
4.1 Settings
To start working with the program, first ensure that you have set up your database and imported your ADIF log file via the Settings window. There for you can have more SQLite3 databases in this program (only to not forget to change to the correct one, where you want to store the images) so you can do for different callsigns or sota or pota and so on. When you decide to have more then one database, you can import different adif files in those databases or when you have only one for all logfile you can also use this on different databases, because the data thats in the main window is shown will be automaticly filtered to only show entries which has a image stored. The rest is in the database but will not be shown.
So let´s start from top to bottom main window click on File then on Settings. The Settings Window will open.
Go to the Database section and click on Select Database Button to choose the database you want to work with. If this is the first time you fired up this program click on "New DB" to create a new SQLite3 database, it opens a browser so you can choose the destination of your database and also its name. Hit Save and the path and name of the new database is shown in the database selection.
Then go to the ADIF Log File section and click on Select Button to choose your ADIF log file you wish to import into the new database. If you do this in a month again only the new ones will be written into database (duplicate check).
Finally go to the Export Folder section and click on Select Button to choose the folder where you want to export your QSL card images later if necessary.
After setting all paths, click on "Cancel Button" to leave the window, changes will be automaticly saved.
Now you are back in the main window and ready to start importing QSL cards.
You can choose between Single Card Import or Bulk Card Import.
4.2 Bulk Card Import
Lets start with Bulk Card Import first, click on Import, then Bulk Card Import and the Bulk Card Import window opens.
Here there are some things to consider:
The program expects the file names of the QSL card images to follow a specific format,
which is defined by the "AC9HP Downloader program" when downloading cards from eqsl.cc.
The expected file name format is:
Callsign=EA2BHE_VisitorCallsign=OE4VMB_QSODate=20231015_Band=20M_Mode=FT8.png
or Callsign=EA2BHE_VisitorCallsign=OE4VMB_QSODate=20231015_Band=20M_Mode=FT8.jpg The program uses this information to crosscheck with the database and import the cards correctly.
So make sure that the images you want to import follow this naming convention.
In the future versions there will be also support for other file name conventions. When you have the images stored in a folder with the correct file names,
click on the "Select Button" to browse to the folder, where the card images are stored.
Then click on the "Import Button" to import all images from the folder into the database.
Once completed, a message will confirm the successful import, numbers of imports and rejected.
But what happens now in the background? The program will read each image file in the selected folder,
extract the relevant information from the file name, and crosscheck it with the entries in your database.
If a matching entry is found, the program will convert the image into a SQLite3 image format named .blob (yes, really it is named so) and store it in the database associated with that entry.
If no matching entry is found, the program will skip that image and move on to the next one.
This process continues until all images in the folder have been processed.
After the import is complete, you can view the imported cards in the main window of the program.
The database view will show the entries that have associated images, allowing you to easily manage and organize your QSL cards.
4.3 Single Card Import
Now let´s try the Single Card Import, click on Import, then Single Card Import and the Single Card Import window opens.
Here you can import individual QSL cards, which you may have received per mail.
You have to fill in the asked fields for the crosscheck with the database:
- Call Sign: Call sign as used in the QSO.
- QSO Date: Enter the date of the QSO in YYYYMMDD format.
//FIXME: Date Conventions!!! - Band: Enter the band on which the QSO took place (e.g., 20M, 40M).
- Mode: Enter the mode of communication used (e.g., FT8, SSB).
Click on the "Select Button" to browse to the image file of the card.
Then click on the "Import Button" to import the image into the database.
Once completed, a message will confirm the successful import or if the card could not be matched.
In the background, the program will read the selected image file, and procced like in the bulk import. After the import is complete, you can view the imported cards in the main window of the program.
The database view will show the entries that have associated images, allowing you to easily manage and organize your QSL cards.
4.4 Search and showing QSL Card Images
In the main window, you can use the search function to find specific QSL cards based on various criteria.
You can enter search terms in the search field and click on the "Search Button" to filter the database view.
The results will be displayed in the database view, showing only the entries that match your search criteria.
You can click on any entry in the database view to see a preview of the associated QSL card image in the Image Preview section (if no image is selected there is a default image shown).
If you want to clear the search and view all entries again, simply click on the "Reset Button".
When you searched for a specific entry and want to see the image larger or have more than one entry, just mark them all with the "Mark all Button" or click on them in the database view, to select some specific. Then hit the "Show Button" and a larger image viewer will open up and you can click through the images.
4.5 Export QSL Card Images
If you wish to export QSL card images from the database back to .png format,
you can use the export function in the main window.
First, use the search function to filter the database view to show the entries you want to export.
Then click on the "Export Button" to export the images of the displayed entries to the specified export folder.
The program will save the images in .png format in the chosen folder,
using the original file name format for easy identification.
A message will confirm the successful export and the location of the exported images.
5. Troubleshooting
If you encounter any issues while using the Eqsl Program, here are some common troubleshooting steps:
- Ensure that you have set up the database and imported your ADIF log file correctly in the Settings window.
- Verify that the file names of the QSL card images follow the expected format for bulk import.
- Check for any error messages displayed by the program and refer to the FAQ section for solutions.
- If the program crashes or behaves unexpectedly, try restarting it and repeating the action.
6. FAQ
Q: What file formats are supported for QSL card images?
A: The program currently supports .png and .jpg file formats for QSL card images.
Q: Can I use multiple databases with the program?
A: Yes, you can create and manage multiple SQLite3 databases within the program.
Q: How do I update the ADIF log file in the database?
A: You can import a new ADIF log file in the Settings window, and the program will update the database accordingly.
